
# (1)制作アプリのタイトル
社内承認を取得せよ

# (2)制作アプリの説明
・最近、社内承認取得に手こずったので、大変さをじゃんけんで再現したゲームです（笑）<br>
・以下(※)を承認フローとして、各ステップの上司とじゃんけん(=承認依頼)を行い、全員に連続で勝利(=承認取得)するとクリアです。負けると、再び課長からになります（１からやり直し）。<br>
(※)課長 → 次長 → 部長 → 管理部長 → 本部長 → GCEO → 社長室会<br>
・救済イベントもつけました。管理部長以上のステージで3回負けると、部長が救済(根回し)してくれて、次回から管理部長とのじゃんけんから始まるようにしてます。<br>
<br>
https://renren-geek.github.io/ren-takanashi-website002/

# (3)工夫した点・こだわった点
## 工夫した点
・承認フローをステージ制で表現した点： 「今どの上司に承認依頼しているか」を番号で管理しました(0=課長、1=次長、2=部長・・・)。 勝ったら次の上司へ進み、負けたら課長に戻るように作りました。<br>
・救済イベントのロジック：カウンターを使って、管理部長以上の上司（Index が3 以上）で負けたときだけ+1するようにしています。この値が3になった時にだけ“救済イベント“が出るように工夫しました。以降は管理部長(index=4)からスタートできるようにしました。

## 難しかった点・次回トライしたいこと（または機能）
### 難しかった点
・条件分岐の整理：自分で条件分岐を複雑にしたのが悪いですが、負けたときに発生する動きが「課長に戻る」「管理部長に飛ぶ」「救済イベントを出す」の3種類あったので、どの順番で if文を書けば正しく動くのかを整理するのに時間がかかりました。

### 次回トライしたいこと
・勝利・敗北時に音を出したいです（BGMを派手にして臨場感出したいです）<br>
・各上司の専用セリフを追加して、キャラ付したいです（余談ですが、この前某上司から「この表現は平安時代の日本語です。」と言われ差戻しされました笑）<br>
・別シリーズも作りたいです（海外場所⇆高梨⇆本部長で板挟みとなり、双方からガン詰めされる等）

# (4)備考（感想、シェアしたいこと等なんでも）
・人的には、①状態管理（どんな値を持つのか/いつ増減・リセットするのか）と、②条件分岐の優先順位付け（最優先で処理すべき条件は？ → 次点の条件は？ → それ以外のデフォルト）の2点が肝だと感じました。<br>
・実装前に、まず①②を紙に書き出して「状態がどう遷移するか、どの条件がどれより優先されるのか」を整理してからコーディングすると、想定外が減ってスムーズに進められると思いました。
